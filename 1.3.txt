Явная проблема с пониманием индексации массивов.
Прономеруем циклы (1-4)
1 - т.к. массивы индексируются с 0, то надо начать именно с 0: (хоть и допускается начинать считывать с 1, но делать на один элемент больше в массиве и 0-ой оставить пустым)
for (int i = 0; i < n; i++)
2 - так же проблема с индексацией:
for (int i = 0; i <= 1; i++) // первый элемент 0-ой, второй 1-ый.
3 - такая же проблема:
for (int i = 8; i > 1; i++) выведем с 8 по 1 (не вкл 1). элемент с инд 8 будет 9ым по счету. это как раз то, что нужно.
4 - for (int i = 9; i < n; i++) 9ый индекс - 10ый по счету. "<= n" ---> "< n". так как n-го элемента в массиве нет. мы его не считывали. даже по 1 циклу видно.

последнее несущественное замечание:
в условие указано, что массив из 15 эл-тов. а реализовано так, что n - размер массива подается на вход.

#include <iostream>
#include <vector>

using namespace std;

int main()
{
    int n;
    cin >> n;
    vector < int > mas(n);
    for (int i = 0; i < n; i++)
    {
        cin >> mas[i];
    }
    for (int i = 0; i <= 1; i++)
    {
        cout << mas[i] << " ";
    }
    for (int i = 8; i > 1; i--)
    {
        cout << mas[i] << " ";
    }
    for (int i = 9; i < n; i++)
    {
        cout << mas[i] << " ";
    }
}
